# STEALTHWATCH-ZERO: Detecting Adversarially-Obfuscated Malware C2 Beaconing via Socket-Level Temporal Entropy Analysis

**Course:** CS362IA — Network Programming and Security, Semester VI  
**Category:** Professional Core Course (Theory and Lab)  
**Research Type:** Experimental + Analytical | Publishable at IEEE/ACM/Springer venues

---

## 1. The Problem

Modern malware communicates with its Command-and-Control (C2) servers using **beaconing** — periodic callbacks over encrypted TLS/HTTPS sockets. Current detection tools (JA3/JA3S TLS fingerprinting, flow-based ML classifiers) are increasingly being evaded by sophisticated adversaries who:

1. **Mimic legitimate traffic fingerprints** (spoofing JA3 hashes of browsers),
2. **Randomize beacon intervals** to defeat periodicity detectors, and
3. **Pad packets** to uniform sizes to defeat payload-length classifiers.

The result: a **detection gap** where encrypted C2 channels blend perfectly into normal HTTPS traffic, and existing ML classifiers fail under adversarial perturbation.

**The core unresolved tension in the literature (2021–2025):** Adversarial evasion attacks on network traffic classifiers have shown that injecting noise into timing and length features can reduce classifier accuracy from >95% to below 40%. Yet no published system defends against this at the **socket-behavior level** — the lowest observable layer before full packet capture is needed.

---

## 2. The Insight (Why This Is Novel)

While C2 detection work focuses on **what** is transmitted (payloads, fingerprints) or **how much** (volume), almost no work rigorously studies **the second-order temporal structure** of TCP socket write/read sequences — that is, the entropy patterns in *inter-write timing deltas* at the socket API call level.

Our key insight:

> **Even when an adversary randomizes beacon intervals at the application layer, the socket-level write cadence carries residual entropy signatures that are structurally distinct from organic human-generated traffic — and this is mathematically quantifiable using Approximate Entropy (ApEn) and Sample Entropy (SampEn) metrics.**

Human-driven sessions (browsing, streaming, video calls) produce high-complexity, context-sensitive write patterns. Automated beaconers — even randomized ones — produce lower-complexity patterns because they are generated by finite-state programs operating under timing budgets. This difference survives adversarial perturbation because changing the entropy signature fundamentally changes the malware's callback behavior in ways that break C2 protocol reliability.

## 3. What Currently Exists (Related Work)

| Work | Approach | Limitation |
|------|----------|------------|
| JA3/JA4 TLS Fingerprinting (2017–2024) | TLS handshake field hashing | Trivially spoofable; evasion tools exist |
| BotFP (IEEE 2020) | Frequency histograms of protocol attributes | No temporal entropy; not adversarially robust |
| MalDIST / CNN-based ETC (2023–2025) | Deep learning on flow features | Degrades ~55% under adversarial perturbation |
| Netskope C2 Beaconing (2024) | Multi-signal anomaly detection (layer 3–7) | Closed commercial system; no formal guarantees |
| NetMasquerade (arXiv 2024) | Black-box adversarial evasion attack on NIDS | Demonstrates the gap; proposes no defense |
| Covert Channel Detection (Wiley 2025) | ML/DL on network timing channels | Covert channel, not C2 beaconing; no entropy formalism |

**The gap:** No published system applies **information-theoretic temporal entropy analysis** at the socket-call level as a primary detection feature that is, by construction, resistant to adversarial timing perturbations.

## 4. The Project: STEALTHWATCH-ZERO

### 4.1 One-Line Description

A **socket-level passive monitor** that computes Approximate Entropy (ApEn) and Sample Entropy (SampEn) of inter-write timing delta sequences on TCP connections, and classifies connections as benign or C2 beaconing using a mathematically grounded threshold derived from the theory of complexity of finite automata.

### 4.2 Core Components

**Component 1 — Socket Tap (Implementation using UNIX Network Programming APIs from the syllabus)**

Using raw sockets or libpcap at the socket layer (connect, send, recv hooks via LD_PRELOAD on Linux), capture:
- Inter-packet arrival time deltas (Δt₁, Δt₂, ..., Δtₙ)
- Packet length sequences (L₁, L₂, ..., Lₙ)

This directly uses `socket()`, `bind()`, `connect()`, `send()`/`recv()`, and `getsockopt()` — core Unit II APIs.

**Component 2 — Entropy Feature Engine**

For a sliding window W of the last N timing deltas, compute:

- **Approximate Entropy ApEn(m, r, N):** Measures regularity/predictability of time series. Low ApEn = high regularity = automated process.
- **Sample Entropy SampEn(m, r, N):** Improved version of ApEn that is less biased for short sequences.
- **Normalized Permutation Entropy (NPE):** Captures ordinal patterns in timing sequences.

These three metrics form a 3-dimensional feature vector per connection window.

**Component 3 — Theoretical Bound (This makes it publishable)**

Derive a lower bound on the minimum entropy a randomized beaconer *must* produce to maintain protocol-reliable C2 communication (keep-alive timing, retry logic, jitter budget). This bound is derived from the Shannon entropy of the program's timing state machine and shows that adversaries face a fundamental trade-off: increasing randomness to evade detection degrades C2 reliability. This is the **novel mathematical contribution**.

**Component 4 — Classifier**

A lightweight One-Class SVM trained only on benign traffic (no need for labeled malware samples). Connections whose entropy vector falls outside the benign region are flagged.

**Component 5 — Adversarial Robustness Evaluation**

Evaluate against:
- Randomized jitter beaconers (simulated)
- Cobalt Strike malleable C2 profiles (public datasets)
- White-box attacks that attempt to maximize entropy while maintaining beaconing

## 8. Why This Satisfies All Conditions

**Solves a Real Problem:** Active gap — adversarial evasion of C2 detection causes millions in enterprise breaches annually. Cobalt Strike C2 evasion was used in SolarWinds, Kaseya, and numerous 2024 ransomware campaigns.

**Highly Novel:** No published work applies information-theoretic entropy formalism at the socket API call level as an adversarially-robust C2 detection primitive.

**Clear Scope:** Input = raw TCP socket traffic. Output = binary classification (benign/C2) + entropy score. No DPI, no decryption.

**Research Continuability:** The theoretical bound extends to QUIC/HTTP3 (post-quantum C2 evasion is an open problem). The framework generalizes to any covert timing channel detection.

**Publishable:** Suitable for IEEE Transactions on Information Forensics and Security, ACM CCS Workshop on Cyber-Physical Threats, or Springer Journal of Cybersecurity. The theoretical contribution distinguishes it from pure empirical papers.

**Mathematically Superior:** The formal entropy lower bound is a verifiable claim — unlike heuristic classifiers, the model has a proof of adversarial resistance under defined conditions.

**Not Overkill:** The core implementation uses standard socket APIs (entirely within CS362IA syllabus), one classical entropy measure, and one well-understood ML primitive (One-Class SVM). No LLMs, no transformers, no distributed systems.

**Directly Uses the Syllabus:**
- Unit I: TCP socket internals, byte ordering, port numbers → socket tap implementation
- Unit II: socket(), connect(), send(), recv(), fork() for concurrent monitoring
- Unit III: UDP analysis for comparing UDP vs TCP beaconing entropy
- Unit IV: RSA/DH key exchange analysis → understanding what TLS hides and what remains visible
- Unit V: TLS/HTTPS traffic analysis → the exact threat model

STEALTHWATCH-ZERO is a focused, implementable, and mathematically grounded research project that:

1. Uses socket-level temporal entropy — a blind spot in all current C2 detection literature
